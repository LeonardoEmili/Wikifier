   Icon is a  |very_high-level_programming_language|Very_High-Level_Programming_Language|  featuring  |goal-directed_execution|Goal-Directed_Execution|  and many facilities for managing  |strings|String|  and textual patterns. It is related to  |SNOBOL|Snobol|  and SL5, string processing languages. Icon is not  |object-oriented|Object-Oriented| , but an object-oriented extension called Idol was developed in 1996 which eventually became  |Unicon|Unicon| .       The Icon language is derived from the  |ALGOL|Algol| -class of  |structured_programming|Structured_Programming|  languages, and thus has syntax similar to  |C|C|  or  |Pascal|Pascal| . Icon is most similar to Pascal, using : syntax for assignments, the procedure keyword and similar syntax. On the other hand, Icon uses C-style brackets for structuring execution groups, and programs start by running a procedure called main .     In many ways Icon also shares features with most  |scripting_languages|Scripting_Language|   .
